# 本次更新记录：患者数据采集与数据库重构

> 用途：记录本轮功能调整涉及的主要内容和文件位置，便于提交代码、说明变更范围和后续回溯。

## 一、权限层级调整

本轮将系统内部权限层级统一为 4 类：

- `user`：患者
- `doctor`：医生
- `manager`：主管
- `admin`：管理员

原来的 `dept_doctor`、`dept_manager`、`campus_manager` 等旧角色口径已逐步替换为新口径。

涉及文件：

- `ruoyi-admin/src/main/java/com/ruoyi/web/controller/system/SysUserController.java`
- `ruoyi-framework/src/main/java/com/ruoyi/framework/web/service/SysRegisterService.java`
- `ruoyi-ui/src/views/system/user/index.vue`
- `sql/0_整合脚本.sql`
- `sql/2_基础运行数据.sql`

## 二、数据库结构重构

新增“账户层 / 患者主档 / 就诊记录 / 医院病历映射”的数据结构，便于支持同一患者多次复诊、多医院、多病历号。

主要变化：

- 新增 `patient_profile`：患者主档，保存姓名、性别、出生日期、监护人、手机号、关联账号等。
- 新增 `patient_chart`：患者在不同医院下的病历号映射。
- 调整 `patient_visit_info`：增加 `patient_id`、`chart_id`、`fill_time`。
- `fill_time` 中文名为“填表时间”。
- 复诊唯一性调整为：同一医院、同一病历号、同一就诊时间、同一填表时间才视为重复。
- SQL 文件整合为 `0/1/2/3` 初始化口径，删除旧的 `4/5` 扩展脚本。
- `sys_role_menu` 角色菜单初始化改为 `NOT EXISTS` 防重复插入，避免初始化时出现重复主键 warning。

- `sql/0_整合脚本.sql`

- `sql/3_业务.sql`
- `sql/README_SQL.md`

## 三、患者账号自动创建与记录绑定

患者账号规则：

- 患者直接填写或医生导入时，系统会按手机号自动创建账号。
- 账号：手机号。
- 初始密码：`123456`。
- 患者角色：`user`。

记录绑定规则：

- 创建账号后，同步生成或更新 `patient_profile`。
- 每条就诊记录写入 `patient_id` 和 `chart_id`。
- 患者登录后查询数据，不再按 `create_by` 判断，而是按当前账号绑定的 `patient_profile.patient_id` 查询。
- 医生导入的患者记录，也会出现在患者自己的数据查询中。

涉及文件：

- `ruoyi-system/src/main/java/com/ruoyi/patient/controller/PatientVisitInfoController.java`
- `ruoyi-system/src/main/java/com/ruoyi/patient/service/impl/PatientVisitInfoServiceImpl.java`
- `ruoyi-system/src/main/resources/mapper/patient/PatientVisitInfoMapper.xml`
- `ruoyi-system/src/main/java/com/ruoyi/patient/domain/PatientVisitInfo.java`
- `ruoyi-system/src/main/java/com/ruoyi/patient/mapper/PatientVisitInfoMapper.java`

## 四、患者公开填写页优化

患者可以无需注册直接填写。

主要变化：

- 提交成功后显示“提交成功”，并进入文档式详情页。
- 增加“返回首页”按钮。
- 患者填写成功后自动生成账号。
- 公开提交接口支持匿名访问，不再因为没有登录态导致写入失败。
- 移动端提交增加事务保护，避免出现“账号创建成功但就诊记录失败”的半截数据。
- 图片未上传时详情页显示“无”。
- 详情页采用文档式展示。

## 六、医生导入与导入失败明细优化

导入优化：

- 医生导入患者数据时，会同步按手机号创建患者账号。
- 导入数据支持“提交答卷时间”列，进入系统后字段名为“填表时间”。
- 如果就诊时间相同但填表时间不同，允许作为不同记录导入。

导入失败明细优化：

- 导入失败明细不再下载易乱码的 CSV。
- 改为下载 Excel 兼容格式，避免中文乱码和行列错乱。
- 失败明细包含行号、病历号、失败原因。
