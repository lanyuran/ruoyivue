# 下一步修改计划

本文档用于确认下一阶段准备实施的修改内容。  
除本计划文档外，暂不继续执行新的业务改动，待你确认后再开始。

## 1. 修改目标

根据 `杨伟吉_原文.docx` 与 `改进要求.txt`，下一步准备重点完善以下内容：

1. 统一前端页面中文文案，避免英文、乱码、表达不一致的问题。
2. 进一步核对“数据上传”和“数据查询”两类页面是否严格符合业务权限要求。
3. 排查并修正“医生/主管只能查看本院数据”的最终生效链路。
4. 排查并修正“院区主管级看不到系统管理页面”的问题。
5. 补一份面向当前项目状态的改动说明，方便后续继续维护。

## 2. 已确认现状

当前项目中，以下内容已经部分具备：

1. 首页已经改成“使用说明”形式，不再展示首页统计数据。
2. 注册页已经支持“患者账号 / 医生账号”两种身份。
3. 医生注册后走审批流程的后端逻辑已存在。
4. 病例详情页已经是偏文档化、适合打印的展示方式。
5. 后端已经存在按医院 `hospitalDeptId` 做数据范围过滤的实现。

当前仍然存在或疑似存在的问题：

1. 部分前端页面文案仍未完全中文化。
2. 数据上传页与数据查询页的职责边界还要继续核对。
3. 医生 / 院区主管的数据范围效果，仍依赖医院树、部门归属、历史数据回填是否正确。
4. 系统管理菜单显示问题，可能是角色菜单绑定、权限配置、数据权限范围共同导致。

## 3. 下一步准备修改的内容

### 3.1 前端页面中文化整理

准备继续检查并修正以下页面中的英文文案、乱码文案、不统一文案：

1. `ruoyi-ui/src/views/patient/information/index.vue`
2. `ruoyi-ui/src/views/check/information/index.vue`
3. `ruoyi-ui/src/views/patient/mobile/index.vue`
4. 必要时补查路由标题、按钮、表单提示、弹窗提示。

准备修改的内容包括：

1. 搜索条件名称改为中文。
2. 表格列名改为中文。
3. 按钮文案改为中文。
4. 表单校验提示改为中文。
5. 导入/删除/保存成功失败提示改为中文。
6. 排查并清理源码中已经写入文件的乱码字符。

### 3.2 数据上传页职责核对

目标：让“数据上传”页只体现“录入 / 导入 / 管理自己上传的数据”。

准备继续确认并在必要时调整：

1. 当前列表是否只显示本人上传的数据。
2. 新增、修改、删除是否只允许在自己的数据范围内执行。
3. 导入后的返回提示、失败明细、下载文件名称是否中文化。
4. 上传页是否还混入了“公共查询”性质的功能，如果有则拆清职责。

### 3.3 数据查询页权限核对

目标：让“数据查询”页体现“按权限范围查看可查询的数据”。

准备继续检查：

1. 患者级是否不进入公共查询页。
2. 科室医生级是否只能看到本院数据。
3. 科室主管级是否能看到本院数据，并有相应管理能力。
4. 院区主管级是否能看到院区范围数据。
5. 管理员级是否保留全量查看能力。

准备重点排查文件：

1. `ruoyi-ui/src/views/check/information/index.vue`
2. `ruoyi-system/src/main/java/com/ruoyi/patient/controller/PatientVisitInfoController.java`
3. `ruoyi-system/src/main/java/com/ruoyi/patient/service/impl/PatientVisitInfoServiceImpl.java`
4. `ruoyi-system/src/main/resources/mapper/patient/PatientVisitInfoMapper.xml`

### 3.4 本院数据权限问题排查

这一项准备分成“代码逻辑”和“数据配置”两部分处理。

代码逻辑侧准备确认：

1. 当前登录用户角色识别是否正确。
2. 当前登录用户 `deptId` 是否能正确解析到医院节点。
3. 查询时是否正确带入 `hospitalDeptId`。
4. 新增/修改病例时是否正确限制为本院。

数据配置侧准备确认：

1. 用户表中的 `dept_id` 是否配置正确。
2. 部门树中是否真的按医院拆开。
3. 历史病例数据中的 `hospital_dept_id` 是否已回填。

说明：如果最终判断问题主要在数据库配置，不会盲目改代码，会单独形成“需要补数据或执行 SQL”的说明。

### 3.5 系统管理菜单显示问题排查

目标：解决“科室主管级、院区主管级看不到系统管理页面”的问题。

准备检查：

1. 角色是否绑定了对应菜单。
2. 角色是否具备用户管理、医院管理等菜单权限。
3. 前端动态路由是否因为权限标识不匹配而被过滤。
4. 是否存在旧角色、冻结角色、弃用角色影响当前菜单展示。

准备重点查看：

1. `ruoyi-admin/src/main/java/com/ruoyi/web/controller/system/SysUserController.java`
2. `ruoyi-ui/src/store/modules/permission.js`
3. 后端菜单、角色、角色菜单关系。
